# Contributing

Pull requests are welcome when they improve the public starter kit without exposing private platform details.

Good contributions:

- New webhook consumer examples
- Better signature verification examples
- Clearer docs
- Make, Zapier, Discord, Slack, Sheets, or dashboard templates
- Safety improvements

Do not contribute:

- API keys
- Signing secrets
- Paid signal data
- Internal scan rules
- Proprietary scoring logic
- Broker credentials

Keep examples simple, readable, and safe by default.
